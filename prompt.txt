infoall 편집 프롬프트 안내
=========================

테마별 issueDraft JSON 스키마가 분리되었습니다. 원하는 테마에 맞는 프롬프트 파일을 선택해 AI에게 그대로 전달하세요. 모든 프롬프트는 "JSON 객체 하나만 출력" 규칙과 "문자열은 한 줄" 규칙을 공유합니다.

사용 가능한 프롬프트
---------------------
- `prompt_사건_정책.txt`: 사건/정책 테마. 진보·보수 시각과 생활 영향 섹션을 포함한 기존 구조를 유지합니다.
- `prompt_육아정보.txt`: 연령대별 육아 가이드, 공통 팁, 긴급 연락처 등을 JSON으로 생성합니다.
- `prompt_건강정보.txt`: 치매·자폐·ADHD·허리 통증 등 주요 질환을 포함한 건강 가이드 스키마입니다.
- `prompt_생활정보.txt`: 생활 꿀팁, 추천 아이템, 핫딜, 제휴 노트를 구조화합니다.

공통 작성 원칙
--------------
1. 출력은 issueDraft JSON 객체 단 하나여야 합니다. JSON 외 텍스트, 코드펜스, 주석을 금지합니다.
2. 모든 문자열은 줄바꿈 없이 한 줄로 작성하고, 큰따옴표가 필요하면 `\"`로 이스케이프합니다.
3. 출처(sources 배열)는 참고한 자료를 신뢰할 수 있는 형식으로 남깁니다.
4. 정보가 부족한 필드는 "정보 부족" 또는 null을 사용합니다(각 프롬프트 지침 참고).

프롬프트 파일을 열어 스키마와 세부 규칙을 확인한 뒤, 해당 템플릿에 맞춰 JSON을 생성해 주세요.
